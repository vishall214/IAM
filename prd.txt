Impact: Ensures critical segregation policies are enforced in risk scores

================================================================================
📊 ENHANCED RISK SCORING PIPELINE
================================================================================

Calculation Flow:
  1. Base Components → sensitivity, infrequency, peer_deviation, recency
  2. Temporal Analysis → detect_temporal_anomalies() identifies time patterns
  3. Adaptive Weights → adapt_weights() boosts relevant components
  4. SoD Check → check_sod_violations() adds compliance risk boost
  5. Composite → risk = Σ(component × adapted_weight) + sod_boost
  6. Classification → LOW, LOW-MEDIUM, MEDIUM, MEDIUM-HIGH, HIGH, CRITICAL

Risk Output Structure:
  {
    "user_id": str,
    "permission_id": str,
    "risk_score": float (0.0-1.0),
    "risk_level": str,
    "components": {
      "sensitivity": float,
      "infrequency": float,
      "peer_deviation": float,
      "recency": float
    },
    "adapted_weights": {  # ← IMPROVEMENT 5
      "sensitivity": float,
      "infrequency": float,
      "peer_deviation": float,
      "recency": float
    },
    "sod_violation": bool,    # ← IMPROVEMENT 4
    "sod_boost": float,       # ← IMPROVEMENT 4
    "temporal_anomaly": bool, # ← IMPROVEMENT 6
    "flag": str (optional)
  }

🔧 IMPLEMENTATION DETAILS
================================================================================

IMPROVEMENT 4 (SoD CONSTRAINTS):
  • Method: _check_sod_violations(user_id, current_perm_id, vectors)
  • Returns: Risk boost value (0.0-1.0)
  • When: Applied during risk_scores calculation
  • Output: sod_violation flag + sod_boost in risk objects
  • Extensibility: Add more policies to self.sod_policies list

IMPROVEMENT 5 (ADAPTIVE WEIGHTS):
  • Method: _adapt_weights(...sensitivity_level, infrequency, temporal_anomaly, statistical_outlier)
  • Returns: Normalized weight dict {sensitivity, infrequency, peer_deviation, recency}
  • Config: self.adaptive_weights_config (tunable boost factors)
  • Normalization: All weights normalize to sum=1.0
  • Context: Considers data sensitivity, patterns, and time windows

IMPROVEMENT 6 (TEMPORAL ANOMALIES):
  • Method: _detect_temporal_anomalies(logs)
  • Returns: Set of anomalous timestamps
  • Algorithm: Hourly distribution + 2σ statistical deviation
  • Threshold: avg_hourly + (2 × stddev)
  • Output: temporal_anomaly flag in risk objects
  • Use: Boosts recency weight via adaptive_weights
